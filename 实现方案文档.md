# 计算机面试题整理与训练系统实现方案（FastAPI + Vue）

## 1. 项目目标

构建一个类似“背单词”的面试题学习系统，支持：

- 本地结构化题库（题目、分类、公司、岗位等）
- AI 批量导入题目（从面经原文自动抽取为结构化 JSON）
- AI 判题与训练反馈（主观题/简答题）
- 学习掌握度量化（单题、分类、全局进度）
- 抽题训练、错题本、复习计划等核心学习功能

技术栈：

- 后端：FastAPI + SQLAlchemy + Pydantic + Alembic
- 前端：Vue3 + TypeScript + Pinia + Element Plus（或 Naive UI）
- 数据库：SQLite（本地起步）-> 可平滑迁移 PostgreSQL
- AI：可配置 OpenAI/通义/DeepSeek 等兼容 API

---

## 2. 功能模块设计

### 2.1 题库管理

- 题目 CRUD：新增、编辑、删除、查询
- 多维标签：分类（操作系统/计网/Java/数据库等）、公司、岗位、难度
- 题目来源：手动录入、AI 导入、批量导入
- 去重机制：按“归一化题干 + 分类 + 公司/岗位”生成哈希判断重复

### 2.2 AI 批量导入（核心）

支持用户粘贴面经原文（例如 nowcoder 页面内容），流程如下：

1. 用户提交原文
2. 后端调用 LLM，按固定 JSON Schema 提取多道题
3. 后端校验 JSON（Pydantic）
4. 去重 + 入库
5. 返回导入报告（成功数、重复数、失败原因）

### 2.3 训练与判题

- 训练模式：
  - 顺序刷题
  - 随机抽题
  - 按分类抽题
  - 按公司/岗位抽题
  - 错题重练
- AI 判题（主观题）：
  - 给出评分（0-100）
  - 维度反馈（准确性、完整性、逻辑性、术语）
  - 改进建议 + 参考答案要点

### 2.4 学习量化与进度系统

- 单题掌握度（0-100）
- 分类掌握度（加权平均）
- 全局学习进度（已学题目覆盖率 + 掌握度）
- 复习推荐（基于遗忘曲线 + 近期错误）

### 2.5 错题本与复盘

- 自动收录低分题（如 <60）
- 记录每次作答和判题结果
- 支持“仅看错题”“最近 7 天错题”“某分类错题”
- 提供弱项分析报告

## 3. 数据模型设计（建议）

## 3.1 核心表

### question（题目表）

- id (PK)
- stem: 题干
- answer_reference: 参考答案（可空）
- category_id: 分类 ID
- difficulty: 难度（1-5）
- source_type: 来源（manual/ai/import）
- source_url: 来源链接（可空）
- created_at / updated_at
- content_hash: 去重哈希

### category（分类表）

- id (PK)
- name（如 操作系统、计算机网络、Java、数据库）
- parent_id（可空，支持层级分类）

### company（公司表）

- id (PK)
- name（字节、腾讯、阿里等）

### role（岗位表）

- id (PK)
- name（前端、后端、客户端、测试、算法）

### question_company（题目-公司多对多）

- id
- question_id
- company_id

### question_role（题目-岗位多对多）

- id
- question_id
- role_id

### user（用户表，单机可先简化）

- id
- username
- created_at

### user_question_state（用户-题目学习状态）

- id
- user_id
- question_id
- mastery_score（0-100，掌握度）
- last_reviewed_at
- next_review_at
- wrong_count
- total_attempts
- last_score

### practice_session（训练会话）

- id
- user_id
- mode（random/category/company/role/wrongbook）
- config_json（抽题参数）
- started_at / ended_at

### practice_record（作答记录）

- id
- session_id
- user_id
- question_id
- user_answer
- ai_score（0-100）
- ai_feedback_json
- is_correct（可按阈值判断）
- created_at

### import_task（导入任务）

- id
- user_id
- source_text
- source_url
- llm_raw_output
- parsed_count
- success_count
- duplicate_count
- failed_count
- status
- created_at

## 4. 掌握度与进度量化方案

## 4.1 单题掌握度（Mastery）

可采用 EMA（指数滑动）更新：

- 新掌握度 = 旧掌握度 * 0.7 + 本次得分 * 0.3
- 本次得分来源：
  - AI 评分（主观题）
  - 或客观题正确率映射到 0-100

惩罚机制：

- 连续答错可额外 -5 ~ -10
- 长时间未复习可按遗忘衰减

## 4.2 分类掌握度

- 分类掌握度 = 该分类下所有题目掌握度的加权平均
- 权重建议：`log(尝试次数 + 1)`，避免新题权重过低

## 4.3 总学习进度

建议拆成两个指标：

1. 覆盖率：已练习题数 / 题库总数
2. 熟练度：所有题目的平均掌握度

总进度可组合为：

- `总进度 = 覆盖率 * 40% + 熟练度 * 60%`

## 5. AI 导入设计（重点）

## 5.1 输入与输出

输入：

- 面经原文（可非常长）
- 可选上下文（公司、岗位、来源 URL）

输出（严格 JSON）：

- questions: 数组，每道题包含：
  - stem
  - category_name
  - companies
  - roles
  - difficulty（1-5）
  - answer_reference（可空）
  - confidence（0-1）

## 5.2 Prompt 策略

- 系统提示词定义“你是结构化信息提取器”
- 强约束“仅输出 JSON，不得输出解释文字”
- 给出 JSON Schema 示例
- 要求：
  - 一段面经可提取多道题
  - 重复题合并
  - 分类名标准化（例如“计网”->“计算机网络”）

## 5.3 鲁棒性处理

- 先尝试 JSON 直接解析
- 失败则进行二次修复（如提取 JSON 子串、修复尾逗号）
- 最终由 Pydantic 校验
- 不合格条目写入 failed list，不影响其余条目入库

## 6. AI 判题设计

## 6.1 判题输入

- 题干
- 用户答案
- 参考答案（如有）
- 评分标准（维度和权重）

## 6.2 判题输出

- 总分（0-100）
- 分维度得分：
  - 准确性
  - 完整性
  - 逻辑性
  - 术语规范
- 优点（1-3 条）
- 缺失点（1-3 条）
- 改进建议（1-3 条）

## 6.3 数据回流

- 将 ai_score 写入 practice_record
- 更新 user_question_state（mastery_score、wrong_count 等）
- 命中阈值（如 <60）自动加入错题本逻辑

## 7. 后端 API 草案（FastAPI）

### 7.1 题库相关

- `POST /api/questions` 新增题目
- `GET /api/questions` 条件查询（分类/公司/岗位/关键词）
- `PUT /api/questions/{id}` 更新题目
- `DELETE /api/questions/{id}` 删除题目

### 7.2 AI 导入

- `POST /api/import/preview` 原文提取预览（不入库）
- `POST /api/import/commit` 确认入库
- `GET /api/import/tasks/{id}` 查询导入任务详情

### 7.3 训练与判题

- `POST /api/practice/sessions` 创建训练会话（抽题）
- `GET /api/practice/sessions/{id}/next` 获取下一题
- `POST /api/practice/submit` 提交答案并触发 AI 判题

### 7.4 统计与进度

- `GET /api/progress/overview` 总体进度
- `GET /api/progress/categories` 分类进度
- `GET /api/wrongbook` 错题列表

## 10. 迭代路线（推荐）

### Phase 1（MVP，1~2 周）

- 题库基础 CRUD
- 分类/公司/岗位管理
- 基础训练（随机抽题 + 手动自评）
- 进度统计（覆盖率、练习次数）

### Phase 2（2~3 周）

- AI 导入（原文 -> JSON -> 入库）
- 去重机制 + 导入报告
- 错题本与弱项分析

### Phase 3（2~3 周）

- AI 判题 + 分维度反馈
- 掌握度模型（EMA + 遗忘衰减）
- 分类与全局进度可视化

### Phase 4（持续优化）

- Prompt 优化与模型切换策略
- 支持图片 OCR 导入（截图面经）
- 个性化复习计划（SM2/FSRS）

---

## 11. 风险与关键点

- LLM 输出不稳定：必须有 Schema 校验 + 容错修复
- 成本控制：导入和判题建议异步队列 + 请求限流
- 数据质量：去重与分类标准化是核心
- 判题一致性：固定评分 rubric + 温度参数尽量低
- 隐私与合规：避免存储敏感个人信息；API Key 本地安全存储

---

## 12. 下一步落地建议

建议先落地以下最小闭环：

1. 建库 + 题库 CRUD
2. 训练会话 + 作答记录
3. AI 导入预览（先不自动入库）
4. 确认数据质量后再开自动入库和 AI 判题

当这个闭环跑通后，再接入掌握度和错题本，就能形成完整学习系统。
